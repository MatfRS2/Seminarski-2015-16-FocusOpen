Configure Metadata

Available to: Super Admin Only
[top ^ | close]

This enables Super Administrators to enable/disable metadata fields, specify if they are required or not, whether they appear in the asset detail view and change the field names. If more than one brand is provided, the user must choose which brand they want to customise. Each brand can have a unique set of metadata.

To Configure Metadata
  • Select Admin > Utilities > Configure Metadata
  • The field column shows the name of the metadata field. Click rename to change the name of the field.
  • The description tells you what type of metadata the field is.
  • Catalogue decides whether an item of metadata will be shown in the catalogue area. Some items are either N/A (i.e. they are system generated and always shown) or disabled because they must be specified for every asset.
  • Required specifies whether the a given field is mandatory or not (this is ignored if the field has not been specified as appearing in the catalogue).
  • Display in Asset Detail determines whether the field is shown in the asset detail pop-up window when the user clicks on a search result in the catalogue.
  • Additional copy allows some extra text to be added to the field name (when editing asset metadata).
  • Tooltip appears when the user is edited the metadata field and allows extra explanatory text to be specified.
  • Hide filter search will suppress the filter (advanced search) options from the catalogue.
  • Hide category search removes the category search from the catalogue.
Click save to apply the metadata changes.

Manage File Paths

Available to: Super Admin Only
[top ^ | close]

Manage file paths enables multiple storage points to be set up and an active one to be specified. Each asset is linked to a file path and as one storage facility runs out of space, it is possible to add more and make the new location the default while the existing file path continues to remain active for existing assets. This section is generally only used by engineers.
